// the problem

Samples don't transpose.

You found the perfect bass on Splice. Rich, gritty, alive. But pitch it down a fifth and it turns into mud. Pitch it up and the character vanishes. The magic lives at exactly one note.

🎵 Find perfect Splice sample
↓
🎹 Sounds amazing at C4
↓
😩 Transpose to F3 → sounds like garbage
↓
🔧 Try to recreate manually → 3 hours later...

✨ What if a machine could find the preset?

What if you could feed a sample to an optimizer, and it figures out exactly which synth knobs to turn to recreate that sound? Then you'd have a playable instrument — not a frozen sample.

// the loop

Optimize until it sounds right.

The concept is simple: render the synth, compare to the target, adjust the knobs, repeat. Each parameter is a dimension in the search space. The optimizer explores until the loss hits zero — or close enough for human ears.

🎯 Target Audio (your Splice sample)
↓ extract features
📊 Compare (multi-resolution STFT loss)
↓ how far off?
🧬 Optimizer (CMA-ES / gradient descent)
↓ adjust knobs
🎛️ Synth (Surge XT via DawDreamer)
↓ render audio
📊 Compare again...
🔄 repeat 500×
✅ Best preset found → export .fxp
~ 500 iterations • ~ 2 minutes on a good CPU • fully headless
// the ideas

Nine ways to build this.

From weekend MVP to nuclear-option JAX pipelines. Tap any card to dig deeper.

🔥
The MVP: CMA-ES Preset Matcher
Drop a sample in, get a Surge XT preset out. Evolutionary optimization, no ML training needed. This is a weekend build.
start here CMA-ES DawDreamer Surge XT auraloss
The stack:
🎛️ Surge XT (free, open-source, 200+ params) 🐍 DawDreamer (headless VST rendering) 🧬 CMA-ES (evolution strategy, pip install cma) 👂 auraloss (multi-res STFT loss)
CMA-ES treats the synth as a black box — no gradients needed. It evolves a population of parameter sets, keeping the ones that sound closest to your target. Handles 200+ dimensions gracefully. Yee-King proved this works in his 2011 thesis — we just have better tools now.
⚡
Differentiable Proxy
Train a neural net to emulate the VST. Then optimize through it with gradient descent. 100-1000× faster.
PyTorch MLP proxy big speedup
Generate 50k random patches with DawDreamer, train a small network to predict params → mel spectrogram. Now you have a differentiable synth. Adam optimizer, 2000 steps, done. The proxy training is the upfront cost — but once you have it, every match is near-instant.
☢️
FAUST → JAX Nuclear Option
Write the synth in FAUST, transpile to JAX via DawDreamer. GPU-accelerated, fully differentiable. 90,000× realtime.
wild JAX FAUST GPU
DawDreamer can transpile FAUST DSP code directly to JAX. SynthAX showed that JAX synths run at 90,000× realtime on GPU. Optimization that takes minutes with CMA-ES could take seconds. The catch: you need to write (or find) your synth in FAUST. But FAUST has a huge library of DSP primitives.
🎹
Multi-Note Optimization
Don't just match one note — match across the whole keyboard. C2 through C6. Velocity too.
robustness musical
A preset that sounds right at C4 but wrong at C2 is useless. Sum the loss across multiple MIDI notes and velocities. This is what makes the result actually playable — the whole point of the project.
🧩
Splice-Aware Workflow
Auto-detect the sound type (pad, lead, bass, pluck), pick the right synth archetype, warm-start the search.
UX CLAP embeddings producer-friendly
Use LAION-CLAP to embed the target audio and compare to text descriptions ("warm pad", "aggressive bass"). The top match pre-initializes CMA-ES in the right neighborhood of preset space. Don't search all 200 Surge params for a pad — start in pad territory.
🧠
DDSP as Universal Proxy
Use Google's differentiable signal processing as a physics-informed proxy. Harmonic oscillators + filtered noise can approximate almost anything.
DDSP differentiable elegant
DDSP's harmonic + noise model is physically motivated. Fit it to your target in ~100 gradient steps, then map the learned characteristics (harmonic content, noise coloring, envelope) to VST parameters. Elegant bridge between neural and classical synthesis.
📦
Batch Mode for Sample Packs
Process an entire Splice pack overnight. 100 samples in, 100 presets out. Browse the results in a web UI.
multiprocessing practical
The real workflow isn't one sample — it's "I want the best sounds from this whole pack as playable presets." DawDreamer supports multiprocessing. Run overnight, wake up to a folder of matched presets with A/B previews.
🔌
Neutone Export → DAW Plugin
Export your trained model as a Neutone VST. Run it directly in Ableton. A neural instrument that sounds like your target.
endgame Neutone SDK Ableton
Neutone lets you deploy PyTorch models as VST/AU plugins. Train a DDSP or RAVE model on your matched sound, export via Neutone SDK, load it in Ableton. You've built a custom neural instrument. MIDI in, audio out, real-time.
🌌
Sound DNA — Latent Space Explorer
Embed sounds in a latent space. Explore neighborhoods. Morph between timbres. Find sounds you didn't know you were looking for.
RAVE embeddings creative
RAVE learns a compact latent space of sound. Train it on your sample library, then explore: what's "between" a pad and a lead? What's the neighborhood of your favorite bass? Navigate by dragging through latent space, hear everything morph in real time. Sound design as exploration, not knob-twiddling.

What's already possible.

These tools exist right now. The building blocks are all here — nobody's assembled them into the preset-from-sample workflow yet.

RAVE — Real-Time Audio VAE

IRCAM's variational autoencoder learns a latent space of any sound. Morphing, interpolation, real-time generation. Now has its own VST.

GitHub →

DDSP — Differentiable Synthesis

Google Magenta's core insight: make signal processing differentiable. Train a violin synth on 13 minutes of audio. Backprop through oscillators.

Magenta DDSP →

DawDreamer — Headless DAW

Load VSTs, send MIDI, automate parameters, render audio — all from Python. The engine that makes this whole project possible.

GitHub →

Neutone — Neural Audio in Your DAW

Deploy PyTorch models as VST plugins. The bridge between research and real music production. Free and growing.

neutone.ai →

Surge XT — Our Target Synth

Free, open-source, incredible sounding, 200+ parameters, Linux native. The perfect first synth to reverse-engineer.

surge-synthesizer.github.io →

SynthAX — 90,000× Realtime

Modular synth in JAX. Fully differentiable, GPU-accelerated. The nuclear option for when CMA-ES feels slow.

GitHub →

// the gap

Nobody's built this yet.

AI can generate entire songs from text prompts now. MusicGen, Stable Audio, Udio, Suno — they're all about creation from scratch. But the inverse problem — "I heard this sound, give me the preset" — remains unsolved at a product level.

The research exists. The tools exist. The papers have been written. Someone just needs to assemble the pieces into something a producer can actually use.
